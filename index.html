Project 04 · Featured case study

MMDA Property Rate Management System

A full-stack civic-technology platform designed to digitise property-rate administration for Metropolitan, Municipal and District Assemblies in Ghana.

Project overview

The platform centralises property registration, classification, rate configuration, billing, payment tracking, dispute management, reporting, and field-enforcement activities. It addresses incomplete property records, manual billing, reconciliation difficulties, limited citizen access, and the inability of officers to verify payment status during inspections.

My role

System planning and architecture

Requirements analysis, workflow mapping, database architecture, model design, and access-control planning.

Full-stack implementation

Django backend, frontend interfaces, billing logic, Django Admin customisation, AJAX forms, reporting, and deployment.

Core capabilities

Centralised property registry

Ownership, use, class, building type, rateable value, GPS coordinates, photographs, communities, and unique property references.

Rate-policy configuration

Annual policies by assembly, property use, class, community, building type, impost, minimum charge, and effective dates.

Billing and payment tracking

Automated rate calculations, annual bills, receipts, outstanding balances, and pending, part-paid, paid, or overdue statuses.

Role-specific portals

Administrative dashboards, a citizen portal, and a mobile-friendly field-officer workflow for inspections and enforcement.

Engineering challenges and solutions

  • Complex policy selection: Built a policy engine that evaluates assembly, use, class, community, building type, billing period, and effective dates before choosing the correct active rate.
  • Dependent administrative forms: Implemented AJAX-powered fields that load only valid communities and automatically populate areas assigned to a selected property class.
  • Duplicate and conflicting records: Added database constraints, model validation, and form-level checks to prevent duplicate classes, communities, and overlapping policies.
  • Historical billing accuracy: Created billing snapshots that preserve the policy name, property class, rateable value, and rate impost used when a bill was generated.
  • Accurate balances: Automated principal, minimum charges, arrears, penalties, outstanding balances, and payment-status updates.
  • Production deployment: Configured PostgreSQL, Gunicorn, WhiteNoise, Render, environment variables, HTTPS, CSRF protection, and secure session cookies.

Technology stack

PythonDjangoDjango REST FrameworkHTMLCSSJavaScriptAJAXPostgreSQLOpenPyXLGitGitHubRenderGunicornWhiteNoise

Project impact

The system replaces fragmented and partly manual processes with one auditable workflow. It is designed to improve property-data quality, billing consistency, payment reconciliation, field verification, revenue transparency, and citizen access to property-rate information.

Project 01 · Machine learning

Maize Disease Detection

A mobile-friendly deep-learning application that classifies diseases from maize-leaf photographs.

Project overview

The application allows a user to upload a maize-leaf image and receive a predicted disease classification. It was designed to make early plant-health assessment more accessible to farmers and agricultural experts working in the field.

My contribution

Machine-learning workflow

Prepared the image-classification pipeline and connected the trained Keras and TensorFlow model to the web application.

Web application

Built the Flask interface for image upload, model inference, result presentation, and mobile-friendly use.

Engineering challenges and solutions

  • Limited labelled data: Applied data augmentation and used available open datasets to improve generalisation and support less common disease classes.
  • Field performance: Optimised and compressed the model to reduce inference time and memory requirements on lower-resource devices.
  • Usability: Kept the workflow simple—upload an image, run the prediction, and display an understandable result.

Project 02 · Computer vision

Smart Customer Sentiment Analysis System

A real-time emotion-detection prototype built for Fidelity Bank Ghana and integrated with the bank's CRM.

Project overview

The system uses facial analysis to classify customer emotions such as happy, neutral, or angry. ResNet-50, TensorFlow, and OpenCV power the detection pipeline, while a REST API passes the resulting data to the bank's CRM.

Measured outcome

88% emotion-detection accuracy

Improved through model refinement and expanded training data.

CRM integration

Created a REST API layer that enabled structured data exchange with the existing system.

Engineering challenges and solutions

  • Low initial accuracy: Expanded and augmented the training data to strengthen performance across the emotion classes.
  • Integration friction: Built a custom REST API layer and coordinated the format of data passed into the CRM.
  • Real-time latency: Optimised the inference pipeline and used hardware acceleration to make live processing more responsive.
  • Validation: The prototype passed user-acceptance testing before completion of the internship project.
